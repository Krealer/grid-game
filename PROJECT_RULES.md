# Permanent Project Rules

## Purpose
- This document is the permanent instruction reference for future development in this repository.
- Codex must read this file before making changes and follow it for every update.

## 1. GitHub Pages Compatibility
- The project must always work on GitHub Pages.
- Only static files may be used.
- No backend is allowed.
- No build tools are allowed.
- No package managers are allowed.
- No environment variables are allowed.
- No server-side code is allowed.

## 2. No External Assets
- No images are allowed.
- No audio files are allowed.
- No sprite sheets are allowed.
- No icon libraries are allowed.
- No external fonts are allowed.
- No CDN dependencies are allowed.
- Visuals must be created using HTML, CSS, Canvas, or JavaScript only.

## 3. Responsive Design Requirements
- Codex must always include both mobile and desktop responsiveness.
- Required media query: `@media (max-width: 767px)`
- Required media query: `@media (min-width: 768px)`

## 4. Input Support
- Desktop support must include keyboard controls.
- Mobile support must include touch controls where relevant.

## 5. Technology Restrictions
- Allowed: HTML, CSS, JavaScript.
- Not allowed: frameworks, engines, external UI libraries, CSS frameworks, external rendering libraries.

## 6. Implementation Discipline
- Codex must check this file before making changes.
- Codex must preserve all listed rules in future edits.
- Codex must not introduce features that violate these rules.

## 7. File Purpose
- This file is a permanent instruction document for future development.
